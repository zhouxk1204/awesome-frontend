# vw 布局优缺点

## 📝 概要
vw 是响应式流体布局的重要工具，但存在小数值、边界控制和渲染问题。

## 📌 核心缺点

### 1️⃣ 小数值问题
- 1vw = 视口宽度的 1%，计算结果可能是小数
```css
/* 假设视口宽度 375px */
.box {
  border-width: 1vw; /* 3.75px */
}
```
- 可能导致模糊或锯齿

### 2️⃣ 不适合极小或极大屏幕
- 小屏幕 → 过小  
- 大屏幕 → 过大

### 3️⃣ 不适合像素精度要求元素
- 边框、阴影、图标等仍需 px 或 rem

### 4️⃣ 性能与渲染问题
- 窗口缩放需重新计算，复杂布局可能增加重绘开销

## ✅ 实践建议
- 流体布局：字体、容器宽高 → vw / clamp()  
- 精准元素：边框、阴影、图标 → px / rem  
- 混合使用：
```css
.box {
  border-width: clamp(1px, 0.5vw, 4px);
}
```
