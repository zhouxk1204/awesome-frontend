# 高效反序列化

## 📝 概要

在 Web 开发中，JavaScript 对象常通过序列化（JSON.stringify）存储或传输，再通过反序列化（JSON.parse）恢复对象。  
虽然 JSON.parse 常用，但它在大型数据或特殊类型处理上存在性能和功能限制。本卡片总结高效反序列化的策略。

---

## 📌 核心知识点

### JSON.parse 的局限性

- 性能问题：大型 JSON 解析可能阻塞主线程  
- 数据类型限制：无法直接处理 Date、函数、undefined、NaN、RegExp 等  
- 安全风险：解析不可信 JSON 有安全隐患

### 提升反序列化效率的策略

#### 1️⃣ 使用 reviver 处理特殊类型

```js
const jsonWithDate = '{"name":"张三","birthDate":"2000-01-01T00:00:00.000Z"}';
const objWithDate = JSON.parse(jsonWithDate, (key, value) => {
  if (key === 'birthDate') return new Date(value);
  return value;
});
console.log(objWithDate.birthDate instanceof Date); // true
```

#### 2️⃣ 流式解析大型 JSON

- 使用库如 `oboe.js` 或 `stream-json`  
- 避免一次性加载大数据造成阻塞

#### 3️⃣ 使用二进制格式

- [MessagePack](https://github.com/msgpack/msgpack-javascript)、Protocol Buffers、BSON  
- 优点：更紧凑、更快解析  
- 缺点：可读性差，适合内部通信而非 API

#### 4️⃣ Web Workers 卸载解析

- 避免阻塞主线程  
- 将大型 JSON 解析交给 Worker 执行

#### 5️⃣ 增量解析与懒加载

- 对于特别大的数据集，可按需解析和加载数据块

### 性能对比

```js
function benchmarkParse() {
  const data = { /* 测试数据 */ };
  const jsonString = JSON.stringify(data);

  console.time('JSON.parse');
  for (let i = 0; i < 1000; i++) JSON.parse(jsonString);
  console.timeEnd('JSON.parse');

  const msgpackData = msgpack.encode(data);
  console.time('msgpack');
  for (let i = 0; i < 1000; i++) msgpack.decode(msgpackData);
  console.timeEnd('msgpack');
}
```

- 小数据集 (<10KB)：JSON.parse 足够  
- 中等数据集 (10KB-1MB)：二进制格式如 MessagePack 更快  
- 大数据集 (>1MB)：流式解析或 Web Worker 更优  

---

## ✅ 总结

- 小数据：标准 JSON.parse  
- 大数据：流式解析 / Web Worker / 二进制格式  
- 特殊类型：使用 reviver 或自定义序列化方案
