# TypeScript：类型守卫 (Type Guards)

## 📝 概述
类型守卫是 TypeScript 提供的一种机制，用于 **在运行时对变量进行类型判断**，从而让编译器能够 **收窄变量类型**，保证访问属性或方法时的类型安全。  
核心场景是处理 **unknown、any 或联合类型** 的变量。


## 🔑 核心原理
当你对一个变量做类型检查后，TypeScript 会自动推断该变量在该分支内的类型：
```ts
function printValue(value: string | number) {
  if (typeof value === 'string') {
    console.log(value.toUpperCase()); // TS知道这里是string
  } else {
    console.log(value.toFixed(2)); // TS知道这里是number
  }
}
````


## 常用类型守卫

### 1. 基础类型检查

```ts
function isString(value: unknown): value is string {
  return typeof value === 'string';
}

function isNumber(value: unknown): value is number {
  return typeof value === 'number';
}
```

### 2. 对象属性检查

```ts
function hasProperty<T extends string>(
  obj: unknown, 
  prop: T
): obj is Record<T, unknown> {
  return typeof obj === 'object' && obj !== null && prop in obj;
}
```

### 3. 数组类型检查

```ts
function isArrayOf<T>(
  value: unknown, 
  validator: (item: unknown) => item is T
): value is T[] {
  return Array.isArray(value) && value.every(validator);
}
```

### 4. 自定义类型保护 (User-defined Type Guard)

```ts
interface User { name: string; age: number; }

function isUser(value: unknown): value is User {
  return typeof value === 'object' && value !== null &&
         'name' in value && 'age' in value;
}
```


## ⚡ 使用示例

```ts
function handle(data: unknown) {
  if (isUser(data)) {
    console.log(data.name.toUpperCase()); // 安全访问
  }
}
```


## 🔧 总结

1. 类型守卫可以收窄变量类型，保证类型安全。
    
2. 对 **unknown、any、联合类型** 特别有用。
    
3. 核心模式：
    
    - `typeof`
        
    - `instanceof`
        
    - 属性检查 `in`
        
    - 自定义类型保护 `value is Type`
        
4. 与 `unknown` 搭配使用，可以强制开发者显式判断类型，提高代码可靠性。
